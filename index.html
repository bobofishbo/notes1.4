Now we get to much more complicated encryption that is far harder to break: sha256.
The reason behind the strength of the function is its increased memory of 256 bits which
makes it practically impossible to break brute force (assuming no prior information).

Our goal here is to firstly show the encryption in progress with our implementation of the
function in python and secondly to create a machine learning model through tensorflow that
is able to match a password with its sha256 cipher.

Due to time and computing constraints, our model is unable to make consistent accurate
predictions. However, the base architecture has been established and some of the following
modifications could be made to make it more effective: switching the loss function to
`binary_crossentropy` and using cloud computing to generate larger datasets of password and
cipher pairs. While we intended to allow users to play with our model through this webpage,
tensorflow was unfortunately incompatible with pyscript in our html, so instead we will show
and explain more code, and provide short demos on how the ML model runs in python.
